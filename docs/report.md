# **Теоретический обзор**

## **Что такое программирование и его роль в современном мире**

**Программирование** – это процесс создания инструкций для компьютера с целью решения задач, автоматизации процессов и управления цифровыми системами. Программы преобразуют входные данные в ожидаемый результат, используя формальные правила языка программирования.

**Основные концепции программирования**

- **Алгоритм** – последовательность шагов для достижения результата.
- **Переменные и типы данных** – способ хранения и обработки информации (числа, строки, логические значения).
- **Управляющие конструкции** – механизмы управления потоком выполнения программы: ветвления (*if*), циклы (*for*, *while*), функции.
- **Структуры данных** – способы организации данных (списки, словари, множества).

Программирование лежит в основе всех цифровых технологий: от веб-приложений до искусственного интеллекта.

## **Git и управление версиями**

**Git** – это система контроля версий, предназначенная для отслеживания изменений в файлах и координации работы нескольких разработчиков над проектом. Использование Git позволяет фиксировать историю разработки, работать над несколькими версиями программы параллельно и эффективно обмениваться результатами.

**Основные понятия Git:**

- **Репозиторий** – хранилище проекта и всей его истории изменений.
- **Коммит** – сохранение изменений с указанием автора и комментария.
- **Ветвь (branch)** – параллельная линия разработки, позволяющая добавлять новые функции без влияния на основную версию.
- **Слияние (merge)** – объединение изменений из разных веток.
- **Удалённый репозиторий** – серверное хранилище (GitHub), позволяющее совместно работать над кодом.

Git обеспечивает безопасность данных, независимость от ошибок и возможность командной работы.

## **Базы данных и SQL**

**База данных** – организованная совокупность данных. В реляционных БД информация хранится в связанных таблицах. Для работы с БД используется язык SQL:

- **DDL** (Data Definition Language) – создание и изменение структуры таблиц (CREATE, ALTER);
- **DML** (Data Manipulation Language) – вставка, обновление, удаление, выборка данных (INSERT, UPDATE, DELETE, SELECT).

Основные принципы проектирования: выделение сущностей, определение атрибутов, установление связей (один-ко-многим, многие-ко-многим), нормализация до 3NF.

**Описание деятельности:** Нами был разработан модуль, доступный для менеджеров (только просмотр) и администраторов (полный CRUD). На странице `/orders` отображается таблица всех заказов с информацией: номер, клиент, даты, пункт выдачи, сумма, статус, код получения. Для менеджера доступен только просмотр и изменение статуса (через выпадающий список). Для администратора добавлены кнопки редактирования и удаления, а также возможность создания нового заказа через форму `/orders/create` с выбором клиента и пункта выдачи из выпадающих списков. При просмотре деталей заказа (`order_detail.html`) отображается полная информация: данные клиента, пункт выдачи, состав заказа с ценами и итоговая сумма, которая рассчитывается автоматически на основе цен товаров и скидок. Все операции с заказами защищены проверкой прав доступа на стороне сервера.
